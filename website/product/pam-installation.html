PAM Installation

Install the Prmana PAM module on your Linux server. This is the server-side component that validates OIDC tokens during SSH authentication.

Prerequisites

Before installing Prmana, ensure your server meets these requirements:

  • Linux with PAM support (Ubuntu 22.04+, Debian 12+, RHEL 9+, Rocky 9+, Amazon Linux 2023)
  • OpenSSH server with PAM enabled
  • SSSD configured and connected to your user directory (LDAP, Active Directory, FreeIPA)
  • Network access to your OIDC Identity Provider's discovery and JWKS endpoints (HTTPS)
  • NTP configured -- clock skew greater than 60 seconds will cause token validation failures

You also need an OIDC Identity Provider (Keycloak, Azure AD, Okta, Auth0) with a client configured for the Device Authorization Grant (RFC 8628).

Package Install

Debian / Ubuntu (apt):

curl -fsSL https://get.prmana.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/prmana.gpg echo "deb [signed-by=/usr/share/keyrings/prmana.gpg] https://apt.prmana.com stable main" | sudo tee /etc/apt/sources.list.d/prmana.list sudo apt update && sudo apt install prmana
Copy

RHEL / Rocky / Fedora (yum/dnf):

sudo rpm --import https://get.prmana.com/gpg sudo dnf config-manager --add-repo https://rpm.prmana.com/prmana.repo sudo dnf install prmana
Copy

The package installs both pam_prmana.so and prmana-agent, sets correct file permissions, and runs ldconfig.

One-liner install: For quick evaluation, curl -sSf https://get.prmana.com | sh handles binary deployment, OS-level dependencies, and initial PAM configuration.

Build from Source

Install the Rust toolchain if you do not have it:

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh source ~/.cargo/env
Copy

Clone and build:

git clone https://github.com/prmana/prmana.git cd prmana cargo build --release
Copy

Install the PAM module to the system PAM directory:

sudo cp target/release/libpam_prmana.so /lib/security/pam_prmana.so sudo chmod 644 /lib/security/pam_prmana.so sudo chown root:root /lib/security/pam_prmana.so

PAM Configuration

Edit /etc/pam.d/sshd to add the Prmana module. The key pattern is: Prmana as sufficient (succeeds on valid OIDC token), with pam_unix.so as the fallback for break-glass access.

# /etc/pam.d/sshd # Prmana OIDC authentication (primary) auth sufficient pam_prmana.so config=/etc/prmana/policy.yaml # Fallback to standard Unix auth (for break-glass) auth required pam_unix.so try_first_pass # Standard account/session handling account required pam_prmana.so config=/etc/prmana/policy.yaml account required pam_unix.so session required pam_prmana.so config=/etc/prmana/policy.yaml session required pam_unix.so
Copy

Configure the SSH daemon in /etc/ssh/sshd_config:

# Enable PAM authentication UsePAM yes # Enable keyboard-interactive (OpenSSH 9.0+) KbdInteractiveAuthentication yes # For OpenSSH < 9.0, use: # ChallengeResponseAuthentication yes
Copy

Restart the SSH daemon:

sudo systemctl restart sshd
Copy
Keep an existing session open while testing configuration changes. If PAM is misconfigured, you could lock yourself out. Always test from a second terminal before closing your admin session.

Verify Installation

Confirm the PAM module loads correctly:

# Check the module file exists and has correct permissions ls -la /lib/security/pam_prmana.so -rw-r--r-- 1 root root 2847632 Apr 12 10:00 /lib/security/pam_prmana.so

Test IdP connectivity from the server:

# Verify the OIDC discovery endpoint is reachable curl -s https://your-idp.example.com/.well-known/openid-configuration | jq '.issuer' "https://your-idp.example.com/realms/corp"

Test PAM authentication (requires pamtester):

sudo apt install pamtester sudo pamtester sshd testuser authenticate

Test a full SSH login from another terminal:

ssh testuser@localhost

Check the server-side authentication logs for Prmana events:

journalctl -u sshd --since "5 minutes ago" | grep prmana
Copy
Common issues: If you see "cannot open shared object file", run sudo ldconfig. If you see "invalid issuer", verify that policy.yaml issuer URL exactly matches the IdP's iss claim (including trailing slash if present).
